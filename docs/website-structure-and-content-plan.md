#### Service 1: Process Discovery & Improvement Sprint

**Headline:** "Process Discovery & Improvement Sprint"  
**Tagline:** "4-6 weeks to build an evidence-based improvement and automation portfolio"

**What You Get:**

**Phase 1: Align & Discover (Weeks 1-2)**
- Define scope: one critical business flow
- Extract and analyze event data
- Process mining and visualization
- Identify current-state patterns, bottlenecks, and variants

**Phase 2: Identify Opportunities (Weeks 2-3)**
- Map improvement candidates:
  - Simple process improvements
  - Automation opportunities
  - Agentic AI use cases
- Classify by type and feasibility

**Phase 3: Build the Portfolio (Weeks 3-4)**
- Estimate business value per opportunity
- Assess effort and complexity
- Create value vs. effort matrix
- Prioritize and rank initiatives

**Phase 4: Roadmap & Governance (Weeks 4-6)**
- 3-6 month implementation roadmap
- Define ownership and KPIs
- Establish light governance framework
- Handover documentation

**Deliverables:**
- Process maps and variant analysis
- Improvement opportunity catalog with value estimates
- Portfolio view (prioritized by value)
- Implementation roadmap
- Governance framework template
- Optional: Transition plan to fractional engagement

**Ideal For:**
- Organizations with critical flows that need improvement
- Companies exploring automation and AI opportunities
- Teams with process mining tools seeking actionable insights
- Leadership wanting transparent ROI for improvement initiatives

---

You get a part-time strategic advisor who:

## Technical Requirements

### Platform & Deployment

**Framework:** Next.js with static export  
**Deployment:** Static hosting (Netlify, Vercel, AWS S3 + CloudFront, or standard web server)  
**Build Output:** Static HTML/CSS/JS

### Key Technical Specifications

1. **Static Site Generation (SSG)**
   - All pages pre-rendered at build time
   - No server-side runtime required
   - Fast initial page loads

2. **Contact Form Handling**
   - Form POST to external service endpoint (to be provided)
   - Client-side validation
   - reCAPTCHA or honeypot spam protection
   - Success/error feedback without page reload

3. **Performance Targets**
   - Lighthouse Score: >90 on all metrics
   - First Contentful Paint: <1.5s
   - Time to Interactive: <3s
   - Total page size: <500KB (excluding images)

4. **SEO Requirements**
   - Semantic HTML5
   - Open Graph meta tags
   - Schema.org structured data (Organization, Service, ContactPoint)
   - Sitemap.xml generation
   - Robots.txt
   - Canonical URLs

5. **Analytics**
   - Google Analytics or privacy-focused alternative
   - Event tracking on CTAs
   - Form submission tracking

6. **Browser Support**
   - Last 2 versions of major browsers
   - Progressive enhancement approach
   - Graceful degradation for older browsers

### Content Management

**Phase 1:** Hard-coded content in React components or markdown files  
**Future Consideration:** Headless CMS (Contentful, Sanity, or simple markdown with MDX)
